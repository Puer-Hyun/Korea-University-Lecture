<span style="color: red;">
[14,12]라는 벡터에 softmax를 적용시키면 어떤 값이 나오니?
</span>

softmax 함수는 입력 벡터의 각 요소에 지수를 적용한 다음, 모든 값을 합산하여 정규화합니다. 이를 통해 확률 분포를 생성합니다. `[14, 12]` 벡터에 softmax 함수를 적용하면 다음과 같은 과정을 거칩니다.

1. 지수를 적용합니다: `e^14`, `e^12`
2. 각 지수 값을 합산합니다: `e^14 + e^12`
3. 각 지수 값을 합산한 값으로 정규화합니다: `(e^14) / (e^14 + e^12)`, `(e^12) / (e^14 + e^12)`

이제 실제 계산을 해보겠습니다. 파이썬 코드를 사용하여 softmax를 계산해보면 다음과 같습니다.

```python
import numpy as np

def softmax(x):
    e_x = np.exp(x - np.max(x))
    return e_x / e_x.sum()

vector = np.array([14, 12])
result = softmax(vector)

print(result)
```

결과는 `[0.88079708, 0.11920292]` 입니다. 따라서 `[14, 12]` 벡터에 softmax를 적용하면 `[0.88079708, 0.11920292]`라는 확률 분포가 생성됩니다. 이 결과는 소수점 여섯자리까지 반올림한 값입니다.

<span style="color: red;">
e_x = np.exp(x - np.max(x)) 이건 뭘 한거야? 결과값은 맞는 것 같은데..
</span>
네, 결과값은 맞습니다. 이 코드에서 `e_x = np.exp(x - np.max(x))`는 입력 벡터 x에 대한 softmax를 계산하기 전에 입력 벡터의 최대값을 뺀 다음, 각 원소에 대해 지수 함수를 적용하는 과정입니다. 이렇게 하는 이유는 주로 컴퓨팅 안정성 때문입니다.

지수 함수를 사용할 때, 큰 값에 대해 지수 함수를 적용하면 매우 큰 값이 나오기 때문에 오버플로우(컴퓨터가 표현할 수 있는 범위를 넘어서는 값)가 발생할 수 있습니다. 이를 방지하기 위해 입력 벡터에서 최대값을 빼는 테크닉을 사용합니다.

입력 벡터의 최대값을 뺀 후 지수 함수를 적용하면, 결과적으로 최대 지수 값이 0이 되고, 다른 값들은 음수가 됩니다. 이렇게 하면 오버플로우를 방지할 수 있으며, softmax 함수의 결과에 영향을 주지 않습니다. 이유는 softmax의 결과는 비율에 관심이 있기 때문입니다.

따라서 `e_x = np.exp(x - np.max(x))` 코드는 오버플로우를 방지하고 안정적인 계산을 위해 입력 벡터에서 최대값을 빼는 과정입니다. 이 과정은 결과적으로 softmax 함수의 출력 값에 영향을 주지 않으며, 안정적인 계산을 보장합니다.
